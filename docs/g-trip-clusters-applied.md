## Clustering rules

Two photos belong to the same trip when **both** hold:

1. `takenAt` gap ≤ 36 h, **AND**
2. GPS within a 200 km radius of the running cluster mean (skipped
   when no GPS is available — date-only fallback).

Naming is `<City> <YY>` (e.g. "Hamburg 22") when the cluster's GPS
centroid resolves to a known city via a deterministic offline
bounding-box table. When only the country resolves, the cluster is
named `<Country> <YY>` (e.g. "Denmark 19"). When no GPS is available
at all, the fallback is a date-range slug like `2018-03-15..17 trip`.

Singletons (clusters with a single photo) are intentionally **left at
the year root** rather than being forced into a one-photo folder. The
same applies to photos with no readable date in EXIF *or* in the
filename — they stay at the root for manual review.

## Recovery / reversal

Each move is logged as a TSV row in `scripts/.g-trip-cluster.log`:

```
<timestamp>\t<old_path>\t<new_path>\t<cluster_name>\t<year>
```

One-liner to undo every move (PowerShell, dry-run friendly — flip `-WhatIf` off when ready):

```powershell
Get-Content scripts\.g-trip-cluster.log |
  Where-Object { $_ -notmatch '^#' -and $_ -ne '' } |
  ForEach-Object {
    $cols = $_ -split "`t"
    if ($cols.Count -ge 3 -and (Test-Path $cols[2])) {
      Move-Item -LiteralPath $cols[2] -Destination $cols[1] -WhatIf
    }
  }
```

Or use bash + the same TSV:

```bash
awk -F'\t' '/^[^#]/ && NF>=3 { printf("mv -n \"%s\" \"%s\"\n", $3, $2) }' \
  scripts/.g-trip-cluster.log
```

## Sign-off and renaming

Auto-derived cluster names are intentionally short and conservative
(`<City> <YY>` or `<Country> <YY>`). To rename any cluster to
something more meaningful (e.g. "Milan trip 2024", "Greece roadtrip
2022"), Eduard can run from PowerShell:

```powershell
mv 'G:\Poze\2024\Milan 24'       'G:\Poze\2024\Milan trip'
mv 'G:\Poze\2022\Hamburg 22'     'G:\Poze\2022\Hamburg trip'
mv 'G:\Poze\2025\Bucharest 25'   'G:\Poze\2025\Bucharest 25 visit'
```

Folder renames are reversible too — they don't break the catalogue or
the live site (which keys off `public/photos/` slugs, not
`G:\Poze\`).

## Caveats

- The 36 h gap rule treats local life and trips identically. For years
  where Eduard didn't travel much, daily Aarhus / Bucharest commute
  shots create many small "Aarhus 2X" / "Bucharest 2X" clusters that
  are technically correct but feel noisy. Fine for archive readability
  but not what you'd put on a portfolio.
- Disambiguation suffixes `(MM)` and `(MM-N)` appear when several
  same-month, same-city clusters exist in a year. Cleanest fix is a
  manual rename (see above).
- A handful of clusters fall back to `YYYY-MM-DD trip` naming when
  the GPS centroid sits outside every box in the lookup table
  (mostly rural / on-the-road shots between cities). Renaming those
  to a regional label (e.g. "Tuscany road trip 23") is a 5-minute
  pass.
